Project Design Checklist

Accessibility

WCAG 2.1 AA standard

Check it meets the WCAG 2.1 latest guidelines for AA standard. View the quick reference and a handy checklist.

Check out these posters from the Home Office Digital and how to make your designs more accessible: posters dos and don'ts

Please note there is a difference between ‘normative’ and ‘informative’, when assessing the level of accessibility that goes into a project.

'Normative' is required for compliance whereas 'Informative' is never required for conformance, and is for information purposes only. Read the Accessibility interpretation problem for more information.

Automated tools

Accessibility checks can be automated with tools such as Lighthouse and WAVE.

Additional tools include: Chrome developer tools, A11y, WebAim colour contrast checker amongst others.

Here is a useful article on how GDS test for accessibility.

Manual checks

Automated tools do not replace testing or expertise. Read more about the pros and cons of using tools.

We do two types of UX testing at OCC for accessibility: expert evaluation and user testing. This allows us to pick up problems quickly and fix issues before we test with users.

You can download free screen reader software, I recommend NVDA.

Copywriting

Spellcheck

Check for consistent use of sentence case, proofread, spell and grammar check any text-based content.

Line length

Reading a long line of type can cause fatigue, too short a line and it breaks words and phrases that are meant to be read as a unit. The ideal line length is 45 – 75 characters. Read an article on balancing line length and font size.

Visual hierarchy

Create a visual organisation using headings, taking advantage of scanning patterns such as F-pattern, using contrast to generate interest, colour, size and space. Read more on understanding visual hierarchy.

Design

Consistency

Ensure consistency of layout, behaviours such as buttons, navigation and any other key elements.

Designing using a grid can help ensure your design achieves effective hierarchy, alignment and consistency. Read more about building better UIs with grid based layouts.

UI elements

Reuse elements and patterns such as popualr pattern libraries and UI kits. Follow style guidelines where they already exist or create your own - see examples of styles guides here.

Handover of assets including UI elements, images, icons etc., any interactions and gestures where necessary. Here is a useful guide on succesful design handoffs.

Development

Consistency

Check consistency with the wireframes, icons and favicon (if there is one).

Responsive

Check cross-browser and on real devices where possible. Ensure relative units are used.

Fundamentals of responsive design

Images

Ensure svg is used for images with some exceptions such as WordPress.

Give Feedback

Errors

Check users can easily recover from errors for example by using error messages and clear actions. Read more on failing gracefully.

Waiting times

If your users have to wait ages for the page to load, show them a loader so they are aware that something is happening. Read 3 tips to make your application feel faster.

Actions

Give immediate and clear feedback of successful user's actions. Don't always wait for server response. Perception is not reality and there are useful techniques for making apps feel faster than they actually are. Read more on performing actions optimistically.

Performance

Loading times

Check page speed and loading times and ensure images are optimised. Read more on how to improve website performance and an essential guide on image optimisation.
